2018-05-25 - KYE

Inside BrushaLoader C2

The [CVE-2018-8174](http://cve.circl.lu/cve/CVE-2018-8174 "Circl.lu CVE-2018-8174") is a bug that allows remote code execution in the VBScript Engine. Found exploited in the wild as a 0day via Word documents, [announced by Qihoo360 on April 20, 2018](https://weibo.com/ttarticle/p/show?id=2309404230886689265523 "新型Office攻击使用浏览器“双杀”漏洞"), [patched by Microsoft](https://portal.msrc.microsoft.com/en-US/security-guidance/advisory/CVE-2018-8174 "CVE-2018-8174 | Windows VBScript Engine Remote Code Execution Vulnerability") on May 8, 2018 and explained in details by [Kaspersky](https://securelist.com/root-cause-analysis-of-cve-2018-8174/85486/ "The King is dead. Long live the King! Root cause analysis of the latest Internet Explorer zero day – CVE-2018-8174") the day after. A Proof of Concept for Internet Explorer 11 on Windows 7 has been [shared publicly 3 days ago](https://github.com/smgorelik/Windows-RCE-exploits/tree/master/Web/VBScript), it's now beeing integrated in Browser Exploit Kits. This will replace [CVE-2016-0189][CVE-2016-0189] from july 2016 and might shake the Drive-By landscape for the coming months.
